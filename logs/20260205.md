4번 영역이 Gemini가 생각보다 약해서 재확인을 하는 부분이 꽤 많다.<br>
코드리뷰를 가상으로 할 경우로 걸러지는 문제가 꽤 많아 보이는 코드를 작성한다.<br>
또한 5번에서 생성된 코드가 단번에 이해안되는 것은 어쩌면 그냥 내가 이해속도가 늦는건지, 아니면 학습용인 것인지 헷갈린다.<br>
Claude에 해당 코드를 하나씩 풀고, 실무적인 방향으로 조금 나은 코드로 조정해 준다.<br>
두 ai를 계속 같이 써야 할 수는 없어서, Gemini 학습조건을 재설정 해준다.<br><br>

1. 최대한 실무프로젝트에서 사용하는 props 위주로 설정<br>
2. 개발자와의 협업을 유념한 코드로 생성 (isLoading, defaultValue미존재, 이벤트 복잡한 경우 view에 hook을 넣지 않고, 별도 js로 생성 <br> > 뭐 사실 이건 내가 만드는 파일이 대부분 view역할이기 때문에 js가 생성될 일이 거의 없어보이지만, 인지는 시켜놔야 될 것 같아서..)<br>
3. 재사용 시 별도 className 정의가 필요한 부분을 정의 (실무에서는 이 부분이 별거 없지만 생각보다 커서..)<br><br>

돌아온 답변 중 엥? 하는 부분이 Boolean props에 보통 isSomething 으로 접두사 is 정도를 추가해서 분리해 놓는 편인데,<br>
오히려 is로 인해 헷갈려서 거의 사용하지 않는다고 하는데,...이게 맞나? 아무래도 이건 아닌 것 같은디..<br>
여튼,<br>
실무프로젝트를 무조건 유념하도록 하니, jsx, css 모두 쫀쫀하게(!) 만들어 준다.<br><br>
