# (2026-01-16) style 입히기 - css library
<br>
css library 중 tailwind css 확인<br><br>
css module 방식이 대규모에 어울린다면, css library 방식은 소규모에 어울린다.<br>
디자이너나 퍼블리셔 없는 개발팀이라면 이만한 css 가이드가 없기 때문이지.<br>
결국 css library를 통한 방식도 확인해 본다.<br><br>

css의 역사를 보면 기존의 html에 섞여 쓰던 styled요소를 구조와 스타일의 구별 관리를 위해 탄생되었지만,<br>
이러한 다중 cssClassName을 활용할 것이라면 왜 굳이 library를 쓰는지 나로선 이해가 안된다.<br>
그냥 css-in-js 방식으로 진행해서 관리하는 것과 크게 다르지 않을 텐데...뭐 그러니까 소규모에 어울리는 것이 겠지만..<br>
하지만, 소규모라 하더라도 결국 공통적인 uiux는 관리되기 마련인데.. (이걸 다들 모르더만, 한장 짜리라도 ui가이드가 존재한다는 사실을 말야...)<br><br>

뭐 그렇지만, 현실은 가장 저렴한 비용으로 공통ui 없이 빠르게 화면을 만들고 싶다면 이러한 library를 안쓸래야 안쓸 수 없겠지.<br>
