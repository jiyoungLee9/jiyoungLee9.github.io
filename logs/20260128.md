# (2026-01-28) state 설계
<br>
정신차리고 컴포넌트 작성.<br>
storybook으로 인해 state에 따른 결과를 한눈에 볼 수 있어서 작성 시 state 설계를 고민하기 시작했다.<br>
모든 화면이 그렇겠지만, 단순 UI (data처리 없는) 라면 상관없지만,<br>
분석/설계 단계에서 UI참여자(기획자, 디자이너, 퍼블리셔, 개발자) 는 모두 동일한 state에 대한 정의를 인지하고 시작하여야 한다.<br><br>

간단해 보이는 UI 컴포넌트라도 결국 state를 어디까지 진행하고, 어떻게 처리할 것인지<br>
생성 관리에대한 구체적인 메뉴얼이 필요하다.<br>
이 지점을 간과할 경우 모든 UI산출물은 어그러지고 만다.<br><br>

현재는 혼자서 진행하기에 생성/관리 마저도 임의 처리하지만,<br>
UI참여자가 2인이상이 될 경우, Agile 방식이라면 이 규칙을 서로 어떻게 정의하는 지가 매우 중요하겠지.<br><br>

이 과정은 AI가 들어와도 바뀔 수 없는 부분이야.. <br>
