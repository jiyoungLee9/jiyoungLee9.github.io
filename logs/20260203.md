# (2026-02-03) 효과적인 ai를 이용한 학습채팅
<br>
Gemini를 통해 현재 1개의 채팅창에서 지속적으로 학습진행 하는 것이 효율적인지 알고 싶어 이것저것 물어봄<br>
사용자 상태와 다른 채팅창을 모두 인지할 줄 알았던 채팅창 별로 Context(맥락)을 공유하지 못함을 확인함 > 각 채팅은 각각의 세션으로 나뉘어 있음을 확인함.<br>
물론 공통 사용자 상태는 설정에서 인지되겠지만, 각 채팅내용은 공유하지 않음을 확인함.<br>
3개월 이상 1개의 채팅사용을 권장하지 않음. 서브채팅창을 만들어 인지하는 것을 추천함.<br>
물론 서브채팅창 시작 시 학습 영속성을 위해 학습 프롬프트를 요청해서 시작해야 함.<br><br>

1개의 채팅창을 3개월 이상 사용 시<br>
초반규칙망각, 속도저하, 논리꼬임 같은 것들이 나온다고 한다.<br>
(나름 치명적인 부분이라고 생각했는데, 물어보기전까지는 절대 알려주지 않는다.)<br><br>

추천규칙을 통해 영속성 유지한 서브채팅창을 만들고, 다시 컴포넌트 진행.<br><br>

useState 훅 진행. <br>
storybook updateArgs 전용을 쓰기보다는 react 표준쓰는 방식으로 요청함.<br>
